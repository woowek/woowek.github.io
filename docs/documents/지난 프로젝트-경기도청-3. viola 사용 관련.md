> viola 사용 관련
----
- 이건 viola의 사용법 안내가 아닌 그냥 후기다.
- openstack의 경우 아예 모르니 그냥 넘어간다.

- kubernetes
    * 대시보드나 뭐 그런건 있어야 할거니까 넘어가고
    * 컨테이너 콘솔을 웹에서 컨트롤할 수 있었다. 이건 확실히 좋더라.
    * 로깅 모니터링 기능은 쓸만하긴 한데.. 로그 다운로드 기능의 한계가 있어서..
    * yaml 유효성 검사기능
    * 일부 세부 설정이 불가능했다.
        - 가장 치명적이었던게.. metal LB가 없다. 컨테이너에 IP 고정이 불가능하다. 
        - 잘 안쓰는 기능이긴 한데 egress설정부분이 없다.
        - 잘 안쓰는 기능이어도 별도의 기타 설정 기능이 필요했지 않을까 싶다. 대시보드에서의 조회가 불가능하더라도...

- devops
    * 클러스터 토큰을 등록, 저장소(gitlab, harbor을 등록)
    * 파이프라인 세부 분리
        - 소스 빌드
        - dockerfile
        - k8s 구성요소 연계
            * deployment
            * services
    * 개인적으로 신기했던게 Argocd를 쓰지 않고 viola 내부에서 파이프라인 구성 후 소스 빌드->docker 빌드->docker image push->deployment/service apply 의 전체 과정을 실행해버린다. 무식해보이지만 오히려 괜찮아보였다.

- 기타
    * 관리자와 사용자 기능이 나뉘어있는데, 정작 사용자 기능이 매우 제한적이다. 적어도 컨테이너 로그 조회나 콘솔 접속까진 가능해야하지 않았을까...

- 결론
    * 버그도 많고, 사용에 제약도 꽤나 있던 솔루션이나, 쓸만한 기능도 있는 솔루션이다. 다른 k8s 오케스트레이션 툴은 안써봐서 비교는 못하겠네..
